### LLVM/Clang/LLDB/LLD

本节内容来自 [LLVM Weekly 第 619 期](http://llvmweekly.org/issue/619)，
[Alex Bradbury](https://www.linkedin.com/in/alex-bradbury/) 大哥持续稳定输出。

* 很好的文章： Arm engineers blogged about [their contributions to LLVM 21](https://developer.arm.com/community/arm-community-blogs/b/tools-software-ides-blog/posts/what-is-new-in-llvm-21).

* 看不过来了： Peter Rong posted an RFC on [providing open evaluation and better onboarding for MLGO (Machine Learning-Guided Optimization)](https://discourse.llvm.org/t/rfc-mlgo-open-evaluation-and-new-onboarding-doc-on-llvm/88771), reflecting on the various challenges people face when trying to use or evaluate things like MLGO for register allocation.

* 赞美，收藏： Slides are now available [from the GPU/Offloading LLVM workshop](https://discourse.llvm.org/t/gpu-offloading-workshop-2025-slides/88832).  Additionally, notes have been posted [from the Embedded Toolchain Workshop](https://discourse.llvm.org/t/llvm-dev-2025-embedded-toolchains-workshop-meeting-notes/88830).

* 宋教授太勤奋了： Fangrui Song shared thoughts on a [compact section header table for ELF](https://discourse.llvm.org/t/compact-section-header-table-for-elf/88821).

* 大胆猜测下，什么时候会有基于 LLVM libc 的 Linux 发行版？ printf error handling was added to LLVM's libc.

### RT-Thread 社区

- KurisaW 提出为软件包引入可配置的 CI 编译与硬件测试框架，允许开发者按需勾选驱动模块，并支持硬件测试的自动化流水线 CI。目前提案尚未有人评论。详见: <https://github.com/RT-Thread/rt-thread/issues/10928>。

- 社区提出编写 PR 提交规范文档，统一贡献者提交PR时的相关规范性问题。<https://github.com/RT-Thread/rt-thread/issues/10926>。

- 目前社区针对 PR# 10537 争议性较大，上周社区例会召开后，对大家反馈的问题进行了一个了解，并提出两种优化方案, 详见 <https://github.com/RT-Thread/rt-thread/issues/10927>。

- BernardXiong 提出对基础系统的 Group 进行整理，当前基础系统（Kernel, shell, libc/posix, file system, network stack, device subsystem）的 Group 划分显得很乱，很随意，需要对这些 Group 划分进行整理，变得简单清晰。详见 <https://github.com/RT-Thread/rt-thread/issues/10904>。
