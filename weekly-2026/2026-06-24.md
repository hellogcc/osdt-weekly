### LLVM/Clang/LLDB/LLD

本节内容来自 [LLVM Weekly 第 651 期](http://llvmweekly.org/issue/651)，
[Alex Bradbury](https://www.linkedin.com/in/alex-bradbury/) 大哥持续稳定输出。

* 今年有人计划去美国的 LLVM 开发者大会么？ The call for proposals is now [open for the 2026 US LLVM Developers' Meeting](https://discourse.llvm.org/t/2026-us-llvm-developers-meeting-call-for-proposals/91080), closing on July 14th. The Dev Meeting itself will take place on October 26th-28th in Santa Clara. Additionally the set of workshops [has been announced](https://discourse.llvm.org/t/2026-llvm-developers-meeting-workshops/91101), [registration is already open](https://discourse.llvm.org/t/2026-us-llvm-developers-meeting-registration-now-open/91122), and [applications for student travel grants are open](https://discourse.llvm.org/t/student-travel-grant-for-the-2026-us-llvm-developers-meeting-applications-open/91121).

* 这个技术点很难： Nikita Popov proposed an [allocator provenance model](https://discourse.llvm.org/t/rfc-allocator-provenance-model/91106).  "We model memory allocation as 'creating' provenance and memory deallocation as 'destroying' provenance. This is reasonably well-defined in cases where the implementation of the allocator lives in a different module. However, it becomes unclear what exactly these terms mean when the allocator is visible in the same module. This RFC is an attempt to specify the provenance semantics of allocators in LLVM, and provide the means to correctly handle them even if asymmetrically inlined."

* 国内也有一些伙伴在贡献： Bruno Cardoso Lopes provided another [status update on ClangIR upstreaming](https://discourse.llvm.org/t/clangir-upstream-progress-report-june-2026/91097).  An `-fclangir`-built clang can now compile about 93% of the translation units in the clang build graph (although the correctness of the compiled TUs has not been tested!).

* 招募维护者： Nikita Popov is [querying if anyone is using the IROutliner and would step up to maintain it](https://discourse.llvm.org/t/ir-outliner-status-interest/89672/9), in order to prevent it being removed.

* 支持： Vittorio Burani started an RFC Discussion on [upstreaming support for the RISC-V OpenHW Group Core-V XCV vendor extensions](https://discourse.llvm.org/t/rfc-core-v-xcv-support-for-cv32e40p-clang-builtins-xcvsimd-intrinsics-and-generic-auto-selection/91111) as implemented in the CV32E40P core.
